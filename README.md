What is SceneScribe?
SceneScribe is an AI-powered video captioning tool built for the AMD Developer Hackathon Act II. It takes a video, analyzes its content, and generates captions in four distinct styles:
Formal: A professional, objective, and factual description.
Sarcastic: A dry, ironic, and witty take on the video.
Humorous Tech: Funny captions with programming and technology references.
Humorous Non-Tech: Funny captions with everyday humor and observations.
This project was built as part of Track 2 of the hackathon, which challenges participants to build a video captioning agent that handles diverse video content and multiple caption styles.
What Makes This Project Special
Unlike many hackathon projects that simply extract random frames, SceneScribe uses a more intelligent approach:
Smart Frame Selection
Most video captioning tools take frames at fixed intervals, which can miss important moments or capture multiple identical frames. SceneScribe instead analyzes the video's visual structure. It uses histogram comparisons and motion detection to identify scene changes and action peaks. This ensures the AI sees the most visually diverse moments in the video, capturing the beginning, middle, and end of the action.
Confidence Scoring
Every caption generated by SceneScribe comes with a confidence score. This tells the user how certain the AI is about each description. A high confidence score means the AI clearly saw what it described. A lower score indicates some uncertainty or guesswork. This transparency helps users understand the reliability of the output.
Self Verification
After generating captions, SceneScribe checks its own work. It verifies that each caption is factually consistent with the original video description. If a caption contradicts the scene or has low confidence, the system regenerates only that specific caption rather than starting from scratch. This ensures higher quality and more accurate outputs.
Adaptive Processing
Not all videos are the same. A short, simple video needs fewer frames than a long, action-packed one. SceneScribe automatically adjusts its processing based on video duration and motion intensity. It uses four frames for simple videos and up to twelve for complex ones, balancing speed and accuracy.

How It Works
The project follows a two-stage pipeline.

Stage One: Visual Understanding

The video is downloaded and its frames are analyzed. SceneScribe uses histogram analysis to detect where scenes change and motion detection to find where action happens. The most informative frames are selected and sent to a vision model (Kimi k2p6) which generates a neutral, factual description of what is happening.

Stage Two: Style Generation

The neutral description is passed to a text model (Kimi k2p5 or GLM 5p2) which rewrites it into the four requested styles. Each caption receives a confidence score, and the system verifies that all captions remain factually consistent with the original scene.

Optional audio transcription is also available. If the video contains speech, the system can transcribe it using Whisper and incorporate that information into the captions for even greater accuracy.

Technical Architecture
The project is built on a modular architecture with clear separation of concerns. Here is a brief overview of the main components.

Configuration handles all environment variables, file paths, and feature toggles in one central location.

Video Downloader retrieves videos from provided URLs with retry logic and exponential backoff for reliability.

Frame Extractor is the heart of the project. It handles motion detection, histogram analysis, and dynamic frame budgeting.

Caption Engine manages all AI interactions through Fireworks API, including visual understanding, style rewriting, confidence scoring, and self verification.

Output Writer saves results atomically to prevent corruption.

Task Loader validates input tasks and handles path resolution.

Audio Extractor optionally extracts and transcribes audio.

Technology Stack
The backend is built with Python, using OpenCV for computer vision, FFmpeg for video processing, and the Fireworks AI SDK for model inference. The system is packaged as a Docker container for portability and reliability.

The frontend is a React application built with TypeScript and TailwindCSS. It features a dark, modern interface with glassmorphism design, animated transitions, and a responsive layout. Users can upload videos via drag and drop or paste a URL, and the interface shows real-time processing status before displaying the captions with visual confidence indicators.

How to Run Locally
The project requires a Fireworks AI API key for the AI models. This can be obtained from the Fireworks platform.

Create a .env file in the root directory with the following:
FIREWORKS_API_KEY=fw_your_actual_key_here

Install the required Python dependencies:
pip install -r requirements.txt

Run the main script to process a sample video:
python main.py

How to Run with Docker
The project is containerized for easy deployment and testing.

Build the Docker image:
docker buildx build --platform linux/amd64 -t sllaayyaa/video-captioner:latest .

Run the container:
docker run --rm -v 
docker run --rm -p 8000:8000 -v "${pwd}/input:/app/input" -v "${pwd}/output:/app/output" -e FIREWORKS_API_KEY="fw_your_key" sllaayyaa/video-captioner:latest

For frontend development, navigate to the frontend folder and run:
cd frontend
npm install
npm run dev

The frontend will be available at http://localhost:5173.
